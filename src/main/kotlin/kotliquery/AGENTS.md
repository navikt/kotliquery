# AGENTS.md — kotliquery source (`src/main/kotlin/kotliquery/`)

## Directory Purpose

This is the sole source package. All public API and internal implementation lives here. No subpackages except `action/`.

## File Responsibilities

### Session.kt — The Central Class (~364 lines)

The main entry point for all database operations. Every query flows through Session.

**Public API surface:**
- `run(action)` — 5 overloads dispatching to action types (Execute, Update, UpdateAndReturnGeneratedKey, List, Nullable)
- `single(query, extractor)` → `A?` — fetch one row, respects `strict` mode
- `list(query, extractor)` → `List<A>` — fetch all rows
- `forEach(query, operator)` — streaming row iteration (low memory)
- `execute(query)` → `Boolean` — DDL/general execute
- `update(query)` → `Int` — DML, returns affected row count
- `updateAndReturnGeneratedKey(query)` → `Long?` — insert with auto-generated key
- `batchPreparedStatement(statement, params)` → `List<Int>` — batch positional params
- `batchPreparedNamedStatement(statement, params)` → `List<Int>` — batch named params
- `batchPreparedStatementAndReturnGeneratedKeys(...)` → `List<Long>` — batch + keys
- `batchPreparedNamedStatementAndReturnGeneratedKeys(...)` → `List<Long>` — batch named + keys
- `createArrayOf(typeName, items)` → `java.sql.Array` — delegate to JDBC
- `transaction { tx -> }` — block-scoped transaction with auto-commit/rollback
- `close()` — closes connection

**Internal mechanics:**
- `createPreparedStatement(query)` — builds `PreparedStatement` from `Query`, handles `returnGeneratedKeys` and Oracle driver workaround
- `populateParams(query, stmt)` — binds params via `setTypedParam`, handles both named (`replacementMap`) and positional (`params`) binding
- `unwrap(v)` — **critical** — unwraps `SqlValued<T>` and Kotlin value classes to their raw JDBC values using `kotlin-reflect`
- `setParam(idx, value)` — dispatches to correct `PreparedStatement.setXxx()` based on runtime type (`when` expression with 20+ branches)
- `warningForTransactionMode()` — logs warning if Session (not TransactionalSession) methods called while `transactional=true`

**Constructor parameters:**
| Parameter | Default | Notes |
|---|---|---|
| `connection` | required | `Connection` wrapper |
| `returnGeneratedKeys` | `true` | But `sessionOf()` factory defaults to `false` |
| `autoGeneratedKeys` | `listOf()` | Column names for Oracle driver |
| `transactional` | `false` | Set `true` during `transaction {}` block |
| `strict` | `false` | `asSingle` throws on >1 rows |
| `queryTimeout` | `null` | Seconds, passed to `PreparedStatement.queryTimeout` |

### Query.kt — Immutable SQL Statement (~45 lines)

Data class holding SQL + params. Immutable and reusable.

**Key fields:**
- `statement` — raw SQL with optional `:paramName` placeholders
- `params` — positional params (`List<Any?>`)
- `paramMap` — named params (`Map<String, Any?>`)
- `cleanStatement` — computed: named params replaced with `?` placeholders
- `replacementMap` — computed: `Map<paramName → List<placeholderIndex>>` for binding

**Named param regex**: `(?<!:):(?!:)[a-zA-Z]\w+`
- `(?<!:)` — not preceded by `:` (avoids PostgreSQL `::` cast operator)
- `(?!:)` — not followed by `:` (same reason)
- Starts with letter, then word chars

**Builder methods:**
- `map(extractor)` → `ResultQueryActionBuilder`
- `asUpdate` (lazy) → `UpdateQueryAction`
- `asUpdateAndReturnGeneratedKey` (lazy) → `UpdateAndReturnGeneratedKeyQueryAction`
- `asExecute` (lazy) → `ExecuteQueryAction`

### Row.kt — ResultSet Wrapper (~382 lines)

Data class wrapping `java.sql.ResultSet`. Implements `Sequence<Row>` for iteration.

**Pattern**: Every type has `type(col): T` and `typeOrNull(col): T?` overloads. Both `columnIndex: Int` (1-based) and `columnLabel: String` variants.

**Supported types**: string, any, long, bytes, float, short, double, int, zonedDateTime, offsetDateTime, instant, localDateTime, localDate, localTime, sqlDate, boolean, bigDecimal, sqlTime, url, blob, byte, clob, nClob, array (reified generic), sqlArray, asciiStream, sqlTimestamp, uuid, ref, nCharacterStream, binaryStream, metaData

**Time handling**: All temporal reads go through `sqlTimestampOrNull()` → convert to target type via `ZoneId.systemDefault()`. This means timezone info is NOT preserved in the DB — it's converted to system timezone on read.

**Nullable pattern**: Uses `private fun <A> nullable(v: A): A?` which checks `resultSet.wasNull()` after each get.

### Connection.kt — JDBC Connection Wrapper (~36 lines)

Thin data class around `java.sql.Connection`.

- `begin()` — `autoCommit=false`, `isReadOnly=false` (unless JTA)
- `commit()` — `commit()`, then `autoCommit=true`
- `rollback()` — `rollback()`, then tries `autoCommit=true` (swallows SQLException)
- `close()` — delegates to underlying

### Parameter.kt — Typed JDBC Binding (~41 lines)

- `Parameter<T>(value: T?, type: Class<T>)` — open class carrying value + type info
- `sqlType()` extension — maps Kotlin/Java types → `java.sql.Types` constants for `setNull()`
- `T?.param<T>()` inline reified extension — wraps any value into Parameter, or re-wraps existing Parameter

### SqlValued.kt — Custom SQL Representation (~5 lines)

```kotlin
interface SqlValued<T> { val sqlValue: T }
```

Implement this on enums or classes to control what value gets bound to JDBC. Session.unwrap() checks for this before value class unwrapping.

### HikariCP.kt — Connection Pool Singleton (~51 lines)

- Named pool registry via `ConcurrentHashMap<String, HikariDataSource>`
- `default(url, user, pass)` — creates/replaces pool named "default"
- `init(name, url, user, pass, configFn)` — creates/replaces named pool with optional HikariConfig lambda
- `dataSource(name)` — retrieves pool, throws if absent/closed
- Default config: `cachePrepStmts=true`, `prepStmtCacheSize=250`, `prepStmtCacheSqlLimit=2048`

### TransactionalSession.kt — Transaction Context (~11 lines)

Minimal subclass of Session. Only exists to give a distinct type inside `session.transaction { tx -> }`. Same API, no additional logic. Does NOT pass `queryTimeout` to parent.

### LoanPattern.kt — Auto-Close Helper (~21 lines)

`using(closeable, f)` — executes block, always closes resource in `finally`. Throws `IllegalStateException` if resource is null.

### package.kt — Top-Level Factories (~44 lines)

- `queryOf(statement, vararg params)` → `Query`
- `queryOf(statement, paramMap)` → `Query`
- `sessionOf(url, user, pass, returnGeneratedKey=false, strict=false, queryTimeout=null)` → `Session`
- `sessionOf(dataSource, returnGeneratedKey=false, strict=false, queryTimeout=null)` → `Session`
- `using(closeable, f)` — delegates to `LoanPattern.using`

**Note**: `sessionOf()` defaults `returnGeneratedKey=false`, but `Session` constructor defaults `returnGeneratedKeys=true`. This is intentional.

## Patterns & Conventions

### Adding a New Type to Row

1. Add `type(columnIndex: Int): T` and `typeOrNull(columnIndex: Int): T?`
2. Add `type(columnLabel: String): T` and `typeOrNull(columnLabel: String): T?`
3. Non-null variants call `typeOrNull(...)!!`
4. Nullable variants call `nullable(underlying.getXxx(...))`
5. Add corresponding `setParam` branch in `Session.setParam()` if the type can be a query parameter

### Adding a New Action Type

See `action/AGENTS.md`.

### Modifying Parameter Binding

Touch `Session.setParam()` for new value types. Touch `Parameter.sqlType()` if a new type needs explicit `setNull()` type mapping.
